# RR Landing — Instruções do Projeto

## Workflow de alterações

Após qualquer correção ou alteração no projeto, sempre:

1. Fazer `git add` dos arquivos alterados
2. Fazer `git commit` com mensagem descritiva
3. Fazer `git push` para o repositório remoto

O push dispara deploy automático na Vercel (projeto `rr-landing`).
